## PROJECT OVERVIEW

Build "The Middle East Hustle" (TMH) — a community-driven polling and founder-profile platform for the Middle East. The core mechanic: provocative question → one-tap vote → results locked behind a share gate (share to WhatsApp OR enter email to unlock) → results animate in with country breakdown. Every vote is a distribution event. The share gate is the growth engine.

**One-sentence pitch:** TMH is the digital town square the Middle East has never had — where 400 million people finally have one place to say what they actually think.

**Tagline:** Real People. Real Hustle. Real Change.

## DESIGN PHILOSOPHY

The design language must feel like "The Economist met Wired, both moved to Dubai, and a 25-year-old designer who grew up on Instagram rebuilt it." Editorial authority. Digital energy. Zero corporate sterility.

**Key principles:**
1. **Editorial Authority** — Every page feels curated. Masthead dates the issue. Section headers look like newspaper column titles. No rounded blob UI. Sharp edges, clear hierarchy, confident typography.
2. **Controlled Drama** — Crimson used sparingly but powerfully. It signals urgency and action. Everywhere else is monochrome.
3. **Data as Narrative** — "9,241 people voted" beats "9K votes". Country breakdown shown as a race, not a pie chart. Results animate in like a reveal.
4. **WhatsApp-Native First** — Every shareable element designed for 375px screen in a messaging app preview. If it doesn't read in a WhatsApp preview, it doesn't ship.

## DATABASE SCHEMA — FULL SQL (Supabase PostgreSQL)

### Table 1: polls
```sql
CREATE TABLE polls (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  question_text text NOT NULL,
  slug text UNIQUE NOT NULL,
  category text NOT NULL, -- 'technology_ai'|'culture_society'|'business'|'cities'|'identity'|'economy'|'women_equality'|'environment'
  options jsonb NOT NULL, -- [{text:'Option 1',votes:0},...]
  status text DEFAULT 'draft', -- 'draft'|'live'|'archived'
  poll_type text DEFAULT 'card', -- 'hero'|'card'|'strip'|'hustler'
  framing text, -- 1-sentence editorial context shown under question
  hustler_id uuid REFERENCES hustlers(id),
  sponsored_by text, -- brand name if sponsored
  publish_at timestamptz,
  created_at timestamptz DEFAULT now()
);
```

### Table 2: votes
```sql
CREATE TABLE votes (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  poll_id uuid REFERENCES polls(id) NOT NULL,
  option_index int NOT NULL, -- 0-based index into options array
  user_id uuid REFERENCES users(id), -- nullable (anonymous vote)
  fingerprint text NOT NULL, -- FingerprintJS visitor ID
  country_code char(2), -- ISO 3166-1 alpha-2
  utm_source text, -- 'whatsapp'|'linkedin'|'x'|'qr'|'direct'
  created_at timestamptz DEFAULT now(),
  UNIQUE(poll_id, fingerprint) -- one vote per device per poll
);
```

### Table 3: users
```sql
CREATE TABLE users (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  email text UNIQUE NOT NULL,
  name text,
  country_code char(2),
  referral_code text UNIQUE DEFAULT nanoid(8),
  referred_by uuid REFERENCES users(id),
  tier text DEFAULT 'explorer', -- 'explorer'|'insider'|'ambassador'|'founding_voice'
  insider_active boolean DEFAULT false,
  role text DEFAULT 'user', -- 'user'|'admin'|'hustler'
  stripe_id text, -- Phase 2
  created_at timestamptz DEFAULT now()
);
```

### Table 4: hustlers
```sql
CREATE TABLE hustlers (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name text NOT NULL,
  slug text UNIQUE NOT NULL,
  title text NOT NULL,
  company text NOT NULL,
  city text,
  country_code char(2),
  bio text, -- 100-200 words, editorial voice
  quote text, -- signature quote
  photo_url text, -- Cloudinary URL
  category_tags text[],
  linked_poll_ids uuid[],
  tag_badge text, -- e.g. 'EXIT FOUNDER', 'UNICORN BUILDER'
  verified boolean DEFAULT false,
  pro boolean DEFAULT false,
  founding_member boolean DEFAULT false, -- true for first 102
  claimed_by uuid REFERENCES users(id),
  profile_views int DEFAULT 0,
  created_at timestamptz DEFAULT now()
);
```

### Table 5: hustler_applications
```sql
CREATE TABLE hustler_applications (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  applicant_email text NOT NULL,
  name text,
  title text,
  company text,
  city text,
  country_code text,
  bio text,
  quote text,
  photo_url text,
  impact_statement text,
  social_links jsonb, -- {linkedin, twitter, instagram, website}
  ai_score int, -- 0-100
  ai_status text, -- 'passed'|'conditional'|'not_yet'
  ai_reasoning text,
  ai_checklist jsonb, -- [{criterion, score, notes}]
  editorial_status text DEFAULT 'pending', -- 'pending'|'approved'|'declined'|'revision'
  editor_notes text,
  created_at timestamptz DEFAULT now(),
  reviewed_at timestamptz
);
```

### Table 6: referrals
```sql
CREATE TABLE referrals (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  referrer_id uuid REFERENCES users(id) NOT NULL,
  referred_id uuid REFERENCES users(id) NOT NULL,
  source_platform text,
  created_at timestamptz DEFAULT now()
);
```

### Table 7: newsletter_subscribers
```sql
CREATE TABLE newsletter_subscribers (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  email text UNIQUE NOT NULL,
  beehiiv_id text,
  country_code char(2),
  source text,
  utm_source text,
  unsubscribed_at timestamptz,
  created_at timestamptz DEFAULT now()
);
```

### Table 8: qr_campaigns
```sql
CREATE TABLE qr_campaigns (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  campaign_name text NOT NULL,
  short_code text UNIQUE NOT NULL, -- e.g. 'step26'
  featured_poll_id uuid REFERENCES polls(id),
  location_tag text, -- e.g. 'Dubai / AstroLabs / March 2026'
  scans int DEFAULT 0,
  conversions int DEFAULT 0,
  email_captures int DEFAULT 0,
  active boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);
```

## CORE USER FLOW #1: THE VIRAL VOTE LOOP (from Figma D-02)

This is the most critical flow. If this is broken on mobile, nothing else matters.

```
Step 1: DISCOVER
  → Friend shares TMH poll link in WhatsApp group
  → Link preview shows: question + live vote count + teaser stat

Step 2: ARRIVE (on /polls/:slug or /join for QR)
  → No pop-up. No gate. No login prompt.
  → Just the question, 4-6 answer options, and live vote counter.

Step 3: VOTE
  → User taps one option
  → Confirmation animation: crimson highlight + checkmark (200ms ease-out)
  → Vote counter increments in real-time (Supabase Realtime)
  → Selected option highlights in crimson
  → FingerprintJS visitor ID logged to prevent duplicate votes
  → Country code inferred from IP

Step 4: SHARE GATE (500ms after vote, slides up from bottom)
  → Full results are BLURRED behind the gate
  → Headline: "UNLOCK FULL RESULTS"
  → Subhead: "Share this debate — or enter your email. Either unlocks the full breakdown."
  → 6 share buttons (in this order, WhatsApp is PRIMARY and LARGEST):
    1. WhatsApp (primary, biggest button, green accent)
    2. LinkedIn
    3. X (Twitter)
    4. Telegram
    5. Instagram Story
    6. Copy Link
  → Email field below with:
    - Auto-focus on share gate appear
    - Placeholder: 'your@email.com'
    - Button: 'UNLOCK RESULTS' (not 'Subscribe' or 'Sign up')
    - On mobile: email keyboard opens automatically
    - GDPR notice: 'No spam. Unsubscribe anytime.'
  → Live counter text: "9,241 people already know how this split. You don't."

Step 5: RESULTS (after share OR email entry)
  → Results animate in with staggered progress bars
    (600ms each, 100ms delay between bars, cubic-bezier(0.25,1,0.5,1))
  → Country breakdown reveals (UAE, SA, EG, JO, LB, Other)

Step 6: CROSS-SELL
```

## CORE USER FLOW #3: HUSTLER APPLICATION (from Figma D-03 and D-05)

```
1. User discovers /apply — Headline: "THINK YOU'RE A HUSTLER?"
   Subhead: "We're building the most credible founder directory in the Middle East. Not everyone makes the cut."
2. Fills out application form with fields:
   - Name, Email, Title, Company, City, Country
   - Bio (100-200 words)
   - Signature Quote
   - Impact Statement
   - Photo URL (or upload)
   - Social Links (LinkedIn, Twitter, Instagram, Website)
3. Submits → Confirmation page (/apply/submitted)
   Text: "APPLICATION RECEIVED. Our AI review runs in minutes."
4. Supabase Edge Function triggers on INSERT to hustler_applications
5. Edge Function calls Claude API with TMH scoring prompt + applicant data
6. Claude evaluates 8-point checklist (see below) and returns JSON:
   { score, status, reasoning, checklist: [{criterion, score, notes}] }
7. Score stored back to hustler_applications row
8. Automated email sent via Resend based on score threshold
9. Application appears in admin queue with colour-coded AI score badge
10. Editor reviews and makes final approval/decline decision (AI is advisory only)
```

### AI Scoring Rubric (8 criteria, 100 points total)

| Points | Criterion | Pass Signal | Fail Signal |
|---|---|---|---|
| 0-15 | Real, verifiable impact on people, companies, or industries | Named outcomes: revenue, users, employees, exits, publications | Vague: 'passionate about', 'experienced in' |
| 0-15 | Based in MENA or deep ongoing connection to the region | Current city in MENA, or explicit MENA focus in bio | No MENA connection evident |
| 0-15 | Unique, specific story — not a generic career path | Pivots, failures, non-linear journeys, specific decisions | Standard corporate progression |
| 0-15 | Built, led, or founded something tangible | Company, product, NGO, movement — with specifics | Title-holder only, no evidence of building |
| 0-10 | Quote is original, specific, and memorable — not generic | First-person, specific to their experience | 'Work hard and dream big' or any cliché |
| 0-10 | Public profile is verifiable (LinkedIn, news, company website) | Active LinkedIn with matching details | No searchable public presence |
| 0-10 | Their audience would genuinely engage on TMH | Has followers / community / known in their sector | No evident audience or network |
| 0-10 | Application complete — all fields filled, photo provided | All fields populated, photo URL returns valid image | Missing fields, broken photo, one-line bio |

### Score Thresholds & Automated Emails

| Score | Status | Email Subject | Email Body Summary |
|---|---|---|---|
| **70-100** | PASSED | "Your TMH application is through." | Congratulations. AI review passed. Editorial reviews within 48 hours. |
| **40-69** | CONDITIONAL | "Your TMH application — one more step." | Profile shows promise. Lists specific gaps from checklist. Resubmit in 30 days. |
| **0-39** | NOT YET | "Your TMH application — keep building." | TMH bar is high. On Rising Hustlers watchlist. Reapply in 90 days. |

### Fallback
If AI call fails → application marked 'pending_manual' → Kareem notified via Resend.

## VOTE PROCESSING LOGIC (Supabase Edge Function)

```
POST /functions/v1/submit-vote
Body: { poll_id, option_index, fingerprint, country_code, utm_source }

1. Check UNIQUE(poll_id, fingerprint) — reject if duplicate
2. INSERT into votes table
3. UPDATE polls.options jsonb to increment vote count for selected option
4. Return { success: true, total_votes, option_votes }
```

Rate limit: Max 1 vote per fingerprint per poll per 24h. Enforce at Edge Function level.
