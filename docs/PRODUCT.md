## 1. Product Overview

**The Tribunal** is MENA's first opinion intelligence platform — part editorial, part data engine, part social experiment. It is a product by **The Middle East Hustle (TMH)**, founded by **Kareem Kaddoura** in **2026**.

The platform's tagline and mission statement:

> **"The Region's First Collective Mirror."**

The Tribunal asks the questions nobody else asks, collects anonymous votes from across 19 countries, tracks predictions over time, surfaces the trends reshaping the region, and profiles the people building it.

From the About page:

> "Think of it as the WSJ of MENA opinion — editorial in presentation, ruthlessly neutral in methodology, and built for the 541 million people who live, work, and build in the Middle East and North Africa."

> "Everything on The Tribunal — every debate, every prediction, every trend, every Voice — adds to a living dataset of what the region actually thinks. Not what governments report. Not what Western media assumes. What real people vote for when nobody's watching."

The homepage presents itself in the style of a newspaper of record:

- **Dateline**: "EST. 2026 · ISSUE NO. 001"

The MENA region is the most opinionated, least surveyed region on earth. 541 million people, zero platforms asking what they think.

## 6. The Majlis

The Majlis is a **private encrypted chat** for verified Voices — a gated community with group channels and direct messages.

- **Access**: Invite-only. Registration requires a valid invite token, matching email, and a verified Voice profile. Entry points: Navbar lock icon, Profiles page hero button, individual profile detail links.
- **Authentication**: Email + password registration. Sessions use server-side opaque tokens (`x-majlis-token` header), stored in `localStorage` as `majlis_token`.
- **Encryption**: Messages encrypted at rest with AES-256-CBC (per-message IV). Decrypted server-side before returning.
- **Layout**: Three-panel design — channel sidebar (left), active chat (center), members panel (right). Mobile: slide-out drawer for sidebar.
- **Channels**: Group channels and DMs. Default "General" channel seeded on first access. DM dedup prevents duplicate conversations.
- **Real-time**: Polling every 3 seconds for new messages, channel list refreshes every 10 seconds.
- **Moderation**: Banned users blocked at auth middleware; muted users shown notice with disabled input; unread indicators on channels.
- **Design**: Dark theme with crimson accents and editorial feel.
- **CMS Management**: User management (ban/mute/activate), message moderation under COMMUNITY section.

### Design Principles

- **Dark-mode-first** with warm parchment light mode toggle (persisted in `localStorage`)

- **High whitespace**: generous padding, max-width `7xl` (1280px) content area
- **Uppercase labels**: section headers, badges, and navigation in uppercase with wide letter-spacing (`0.15em`–`0.28em`)
- **Editorial presentation**: datelines, issue numbers, section dividers, newspaper-style column layouts
- **Scrolling tickers**: horizontal scrolling strips for data feeds (debates, predictions, pulse, voices) using `tmh-ticker-scroll` CSS animation
- **Muted foreground** at 63% lightness for secondary text
- **Crimson period**: "The Tribunal**.**" — the period is always in crimson `#DC143C`

- Scrolling tickers (90s linear infinite for fast, 200s for slow)

### AI Ideation Engine

A 5-step agentic workflow for AI-powered content generation:

1. **Research** — Uses Perplexity API to gather current MENA trends and context
2. **Generate** — Uses Claude (Anthropic) to create debate questions, predictions, or pulse topics
3. **Safety Review** — AI risk assessment with flag levels and flag types
4. **Cherry-Pick** — Editor reviews generated ideas, accepts or rejects with tags
5. **Publish** — Accepted ideas are published to the platform

Database tables:
- `ideation_sessions` — Session tracking with config, status, counts
- `ideation_ideas` — Generated ideas with content, risk flags, status
- `ideation_rejection_log` — Rejection tracking with tags
- `ideation_exclusion_list` — Phrases to exclude from generation
- `ideation_prompt_templates` — Customizable prompt templates per pillar type

Falls back to intelligent mock data when API keys are not configured.

Content follows a status-based workflow with controlled transitions:

```
draft → in_review → approved → flagged → archived
                  → rejected → revision → in_review
```

All public API endpoints filter by `editorialStatus = 'approved'` — only approved content reaches the frontend.
